# 컨퍼런스 기획 에이전트 — PRD (Product Requirements Document)

> AI SUMMIT AND EXPO 2026 기획을 위한 AI 에이전트 웹 애플리케이션

---

## 1. 문제 정의

### 현재 상황 (AS-IS)
- 수작업 웹 검색으로 트렌드 파악 → 시간 소요 큼
- **해외 연사 발굴이 최대 병목** — 국내는 네트워크로 찾지만, 해외는 리서치 비용이 높고 후보 풀이 좁음
- 연사 적합성 판단 기준이 기획자의 머릿속에만 존재 (암묵지)
- Google Docs에 흩어진 기획 정보, 미팅 후 구두로 방향 수정
- 일회성 리서치로 끝나고, 시간이 지나면 다시 처음부터

### 해결 방향 (TO-BE)
- AI가 트렌드 리서치 + 근거 자료를 체계적으로 정리
- AI가 해외 연사 후보를 추천하고 프로필·강연 이력·온라인 평판 기반으로 적합성 스코어링
- 한 곳에서 아젠다/연사/피드백을 통합 관리하는 웹 대시보드
- 피드백 입력 후 에이전트가 반영해서 지속적으로 업데이트

## 5. 핵심 기능

### 5-1. 트렌드 리서치 에이전트 (P0)
- **입력**: 산업/도메인 키워드, 지난 행사 데이터
- **처리**: 웹 검색, 해외 주요 컨퍼런스 프로그램 분석 (NeurIPS, ICML, Google I/O, CES, Web Summit 등), 뉴스/리포트 요약
- **출력**: 트렌드 키워드 + 근거 자료 (출처 포함) + 트랙 설계 입력값
- **핵심 가치**: "올해 AI 컨퍼런스에서 가장 많이 다뤄지는 주제"를 데이터 기반으로 분석

### 5-2. 연사 추천 에이전트 (P0) — 킬러 기능
- **입력**: 트랙/세션 주제, 연사 티어 요구사항, 선호 조건
- **처리**:
  - 해외 컨퍼런스 연사 데이터 수집
  - LinkedIn 프로필 탐색
  - 기존 네트워크 + 스노우볼링 (연사 관계 그래프)
  - **적합성 스코어링**: 전문성, 네임밸류, 강연 이력, 온라인 평판
- **출력**: 세션별 연사 후보 리스트 (프로필, 스코어, 추천 이유)

#### 연사 티어링 기준
| 티어 | 세션 | 핵심 기준 |
|------|------|-----------|
| **Tier 1 (키노트)** | 오전 키노트 | 전문성 + 네임밸류 최우선 |
| **Tier 3 (트랙)** | 오후 트랙 발표 | 실무 성과/사례 중심 (AI를 실제 적용해서 성과를 낸 사람) |

#### 연사 소싱 채널
- 해외 주요 컨퍼런스 연사 목록
- LinkedIn 프로필 탐색
- 기존 연사 네트워크
- 스노우볼링 (연사가 다른 연사를 소개)
- 연사 신청 접수
- 스폰서/파트너 유료 세션

### 5-3. 웹 대시보드 (P1)
- 리서치 결과, 아젠다, 연사 후보를 한 곳에서 조회
- 팀원 4명이 함께 보고 피드백 입력
- 상태(state)를 DB에 저장하여 지속적으로 갱신

### 5-4. 피드백 → 재생성 루프 (P1)
- 미팅 후 기획 방향 피드백을 자연어로 입력
- 에이전트가 피드백 반영하여 리서치/아젠다 업데이트
- 예: "이런 아젠다와 연사가 추가로 필요하다", "한국에서 인기 있는 연사 스타일"

### 5-5. 아젠다 구성 에이전트 (P2)
- **트랙 수**: 4개 (매년 트렌드에 따라 재구성, 고정 구조 아님)
- **세션 형식**: 키노트 + 패널 중심
- 트렌드 리서치 결과가 트랙 설계의 입력값

### 5-6. 기획안 문서 생성 (P2)
- 아젠다 + 연사 + 피드백을 종합한 프로그램 기획 문서
- Markdown 출력 (추후 PDF 변환)

## 6. 에이전트 워크플로우

```
[사용자 입력: 테마/키워드/피드백]
        │
        ▼
┌─────────────────┐
│ 트렌드 리서치    │ ← 웹 검색, 해외 컨퍼런스, 이전 행사 데이터
│ 에이전트        │
└────────┬────────┘
         │ 트렌드 키워드 + 근거
         ▼
┌─────────────────┐
│ 트랙/아젠다 구성 │ ← 4개 트랙, 매년 재구성
│ 에이전트        │
└────────┬────────┘
         │ 아젠다 초안
         ▼
   [웹 대시보드에서 팀 리뷰]  ← Human-in-the-loop
         │ 피드백
         ▼
┌─────────────────┐
│ 연사 추천       │ ← 해외 컨퍼런스, LinkedIn, 스노우볼링
│ 에이전트        │     적합성 스코어링
└────────┬────────┘
         │ 연사 후보 + 스코어
         ▼
   [웹 대시보드에서 팀 리뷰]  ← Human-in-the-loop
         │ 피드백
         ▼
┌─────────────────┐
│ 피드백 반영     │ ← 자연어 피드백 → 재리서치
│ + 재생성 루프   │
└────────┬────────┘
         │
         ▼
   [기획안 점진적 완성]
```

## 7. 팀에이전트 아키텍처

### 토론 기반 협업 패턴
에이전트들이 서로 대화하며 결론을 도출하는 구조.
단순 리서치 → 저장이 아니라, **제안 → 비평 → 수정 → 합의** 사이클.

### 기능 에이전트
- **Trend Researcher** — 웹 검색으로 트렌드 수집/분석
- **Speaker Researcher** — 해외 연사 발굴/프로필 수집/스코어링

### 페르소나 에이전트 (토론 패널)
| 페르소나 | 관점 | 핵심 질문 |
|---------|------|----------|
| **AI 기술 전문가** | 기술적 타당성, 최신성 | "기술적으로 충분히 깊은가?" |
| **대기업 참가자** | 비즈니스 가치, ROI | "100만원 티켓 가치가 있는가?" |
| **오퍼레이션 담당자** | 실행 가능성 | "실제로 섭외할 수 있는가?" |
| **일반 참가자** | 흥미도, 접근성 | "이거 재밌나?" |

### Orchestrator
팀 토론을 진행하고 합의를 도출하는 중앙 조율자.

## 9. 성공 기준

| 기준 | 측정 방법 |
|------|-----------|
| **해외 연사 후보 풀 확장** | 기존 수작업 대비 후보 수 3배 이상 |
| **리서치 시간 단축** | 기존 대비 50% 이상 시간 절감 |
| **연사 추천 적합도** | 추천 연사 중 50% 이상이 실제 섭외 검토 대상 |
| **팀 협업 효율** | 기획 정보가 한 곳에서 관리되어 미팅 준비 시간 감소 |

## 10. MVP 우선순위

| 우선순위 | 기능 | 이유 |
|---------|------|------|
| **P0** | 트렌드 리서치 에이전트 | 병목 1순위, 가장 즉시 가치 |
| **P0** | 연사 추천 에이전트 | 병목 2순위, 핵심 pain point |
| **P1** | 웹 대시보드 (결과 조회 + 피드백 입력) | 팀 공유/배포에 필수 |
| **P1** | 피드백 → 재생성 루프 | "지속적 업데이트"의 핵심 |
| **P2** | 아젠다 자동 구성 | 리서치 + 연사 확보 후 상대적으로 쉬움 |
| **P2** | 기획안 문서 자동 생성 | 최종 출력 포맷팅 |
